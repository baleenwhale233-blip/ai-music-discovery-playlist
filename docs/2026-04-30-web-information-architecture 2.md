# Web 端整体信息架构 v0.4

## 1. 本版定位

本文是 `docs/basic_ia.md` 之后的 Web 产品化信息架构补充，聚焦正式 `apps/web` 的登录后首页、底部导航和添加听单流程。

当前真实路线保持不变：

```text
目录
-> 用户主动本地缓存
-> 听单
-> 真播放器
```

本版新增的产品重心是：

```text
登录
-> 听单广场
-> 创建听单草稿
-> 添加视频 / 合集链接
-> 管理待发布条目
-> 发布目录元信息
-> 用户主动缓存
-> 真播放器收听
```

`发布听单` 在本版只表示发布目录元信息，不表示公开音频托管，也不表示共享用户已经缓存的音频文件。

## 2. 核心原则

- 登录后默认进入听单广场，而不是旧的导入工具页。
- 广场展示的是公开听单目录元信息，不是公开播放器。
- 听单可以公开展示标题、简介、封面、来源条目和创建者信息。
- 音频缓存文件仍属于触发缓存的用户个人资产，不跟随听单发布给其他用户。
- 播放仍只发生在用户主动缓存完成之后，继续使用真实 media element / `<audio>`。
- 添加听单从“单页导入工具”升级为“听单草稿创建流程”。
- `apps/web` 只通过 `/api/v1` 正式端点和 `packages/api-contract` 对接后端，不直接引用 `apps/api` 或 `apps/admin` 内部代码。

## 3. 一级导航

Mobile Web 底部固定导航采用三入口结构：

```text
听单     + 添加听单     我的
```

### 3.1 听单

登录后的默认首页。

目标：

- 展示听单广场。
- 让用户发现值得保存的 AI 音乐目录。
- 让用户从公开目录进入详情后决定是否保存为自己的本地音频。

内容：

- 推荐听单模块。
- 最新发布模块。
- AI 音乐专题模块。
- 学习听单模块。
- 用户自己最近发布或最近缓存的听单入口。

### 3.2 添加听单

底部导航中央的大按钮，是当前 Web 端最重要的创建入口。

目标：

- 创建一个待发布听单草稿。
- 让用户逐步添加单条视频、合集、收藏夹或播放列表链接。
- 在发布前完成标题、简介、删除、批量管理和排序。

交互策略：

- 底部中央按钮视觉上比左右导航更突出。
- 点击后进入草稿首页。
- 草稿首页再通过“添加视频”进入二级链接解析页。

### 3.3 我的

个人空间和管理入口。

内容：

- 我的听单。
- 草稿箱。
- 已发布听单。
- 最近缓存。
- 最近播放。
- 缓存中 / 缓存失败。
- 存储空间与清理缓存。
- 来源和实验功能设置。
- Alpha 账号信息。

播放器能力后续可以从“我的听单”或听单详情进入，不再要求底部导航单独放播放器入口。

## 4. 听单广场

### 4.1 页面目标

听单广场的主心智是：

```text
发现值得保存的目录
```

不是：

```text
直接在线播放公开音频
```

用户在广场看到的是别人或编辑发布的听单目录。若想真正收听，需要进入详情并主动缓存到自己的本地音频资产中。

### 4.2 布局参考

视觉结构可以参考用户给出的参考图右侧：多个独立模块以不规则但有秩序的卡片块组成。适合移动端首屏呈现高密度但轻松的内容浏览。

建议布局：

- 顶部：页面标题、搜索入口。
- 主内容：双列或瀑布式模块卡片，卡片高度可根据内容类型变化。
- 底部：三入口导航，中央“添加听单”按钮突出。

首期可以先采用规则双列卡片，后续再根据内容运营能力做更自由的模块布局。

### 4.3 卡片信息

广场听单卡片展示：

- 合集封面或多封面拼图。
- 听单标题。
- 听单简介或短备注。
- 条目数量。
- 来源平台，例如 B 站。
- 创建者 / 作者 / 编辑标识。
- 目录类型，例如 AI 音乐、AI 翻唱、学习、混合。
- 当前用户缓存提示，例如“未缓存”“已缓存 8/24”。

卡片不展示：

- 可直接播放的公开音频地址。
- 其他用户的本地缓存文件。
- 伪播放进度或 iframe 播放器状态。

### 4.4 点击路径

```text
听单广场卡片
-> 听单详情
-> 查看目录条目
-> 选择要保存的条目
-> 用户主动缓存
-> 加入自己的本地听单
-> 真播放器播放
```

## 5. 添加听单流程

### 5.1 草稿首页

入口：底部中央 `添加听单`。

草稿首页内容：

- 听单标题输入。
- 听单简介输入。
- 合集封面预览，可先由条目封面自动生成。
- “添加视频”主按钮。
- 等待保存的听单列表。
- 批量管理入口。
- 排序入口。
- 发布听单按钮。

空草稿状态：

```text
还没有视频
-> 添加视频
```

非空草稿状态：

```text
待保存条目列表
-> 继续添加
-> 删除
-> 批量管理
-> 排序
-> 编辑标题和简介
-> 发布听单
```

### 5.2 二级添加视频页

入口：草稿首页点击 `添加视频`。

页面内容：

- 链接输入框。
- 支持说明：B 站单条、合集、收藏夹、播放列表。
- 解析按钮。
- 解析状态。
- 错误提示。

支持链接：

- B 站单条视频。
- B 站收藏夹。
- B 站播放列表。
- B 站合集 / media list。

首期不在这里做复杂筛选。解析完成后返回草稿首页，由草稿列表承接后续管理。

### 5.3 解析完成后的返回

成功路径：

```text
二级添加视频页
-> 解析完成
-> 自动回到草稿首页
-> 新条目进入等待保存的听单列表
```

失败路径：

```text
二级添加视频页
-> 解析失败
-> 留在当前页
-> 显示失败原因
-> 用户修改链接后重试
```

若一次解析出多个条目，默认全部加入草稿列表，再由用户在草稿首页删除、批量管理或排序。

### 5.4 草稿列表管理

待保存条目支持：

- 单条删除。
- 批量选择。
- 批量删除。
- 批量保留 / 取消选择。
- 调整顺序。
- 查看条目标题、封面、作者、时长、来源链接。

排序是本版新增能力。移动端首期可以先采用：

- 长按拖拽排序；或
- “上移 / 下移”按钮作为保底实现。

### 5.5 发布听单

发布前必须有：

- 标题。
- 至少 1 个条目。

简介首期可以选填，但发布页要鼓励用户填写一句说明，方便广场卡片展示。

发布结果：

```text
听单草稿
-> 发布目录元信息
-> 进入听单广场 / 我的已发布
```

发布不触发：

- 自动缓存音频。
- 公开分享音频。
- 公开播放其他用户的缓存文件。

## 6. 已发布听单与缓存

完成发布后，听单成为目录元信息对象。用户可以对听单里的条目执行本地缓存。

缓存入口：

- 听单详情页。
- 我的听单页。
- 缓存状态列表。

缓存状态：

- 未缓存。
- 缓存中。
- 已缓存。
- 缓存失败。

缓存完成后：

```text
已发布听单目录
-> 用户主动缓存条目
-> 生成个人 LocalAudioAsset
-> 加入个人可播放听单
-> 真播放器播放
```

同一个公开听单可以被多个用户分别缓存。每个用户拥有自己的缓存状态和本地音频资产，不共享转换后的音频文件。

## 7. 页面地图

建议后续 Web 页面结构：

```text
/login

/
  登录后重定向到听单广场

/playlists
  听单广场

/playlists/[playlistId]
  听单详情
  目录条目
  缓存入口
  已缓存后进入播放器

/playlists/new
  添加听单草稿首页

/playlists/new/add
  二级添加视频页

/me
  我的
  草稿箱
  已发布听单
  最近缓存
  最近播放
  设置

/player/[playlistId]
  真播放器页面
  只播放当前用户已缓存成功的音频
```

现有 `apps/web` 的导入页能力后续应迁入 `/playlists/new/add` 和 `/playlists/new`，旧 `/playlist` 播放器页能力后续应迁入听单详情或 `/player/[playlistId]`。

## 8. 后续接口方向

本文不直接修改 API 契约，但后续实现可能需要补充以下正式接口：

- 创建听单草稿。
- 更新草稿标题和简介。
- 解析链接并追加到草稿。
- 删除草稿条目。
- 批量管理草稿条目。
- 调整草稿条目顺序。
- 发布听单目录元信息。
- 获取听单广场列表。
- 获取听单详情。
- 对已发布听单条目发起本地缓存。

接口仍应归属 `/api/v1`，契约先进入 `packages/api-contract`，领域类型先进入 `packages/types`。

## 9. MVP 分期

### Phase A：文档与 IA 锁定

- 新建本文。
- 后续实现前同步更新 `docs/basic_ia.md` 或将本文作为 v0.4 来源。

### Phase B：静态 Web 骨架

- 登录后首页改为听单广场壳。
- 底部三入口导航。
- 添加听单草稿页面壳。
- 二级添加视频页面壳。
- 我的页面壳。

### Phase C：复用现有导入能力

- 将现有链接解析能力迁入二级添加视频页。
- 解析结果回填草稿列表。
- 支持删除、批量管理和排序。

### Phase D：发布与广场

- 新增听单目录元信息发布能力。
- 新增广场列表。
- 新增听单详情。

### Phase E：缓存与播放器整合

- 已发布听单支持用户主动缓存。
- 缓存完成后进入真播放器。
- 我的页面展示缓存状态和失败重试。

## 10. 明确不做

- 不做公开音频托管。
- 不公开分享用户缓存文件。
- 不把 iframe/WebView 播放器作为正式播放器。
- 不把来源视频页包装成主播放体验。
- 不把重社区、评论、关注流作为当前 MVP 主线。
- 当前阶段不创建 `packages/ui`。
