# PRD v0.2：AI 音乐目录与本地听单

## 1. 产品定义

**工作名**：AI 音乐发现与听单

**一句话**：一个面向大陆用户的 AI 音乐目录与本地听单工具。用户可以导入 B 站等来源链接，把内容整理成目录/听单，并在用户主动保存后转成本地音频，用原生播放器连续收听。

## 2. 产品定位

### 我们做什么

- AI 音乐目录
- 来源链接整理
- 用户主动本地缓存
- 本地音频播放器
- 播放单/听单管理
- 轻量 AI 信息展示

### 我们不做什么

- 不做公开音频托管
- 不做用户上传型音乐平台
- 不做评论广场
- 不做重社区
- 不做版权归属认证平台
- 不把本地缓存文件公开分享给其他用户
- 不在首页默认露出高风险或不稳定来源

## 3. 核心变化

旧路线是“可播放目录型工具”，依赖来源站外链播放器。实测后发现 iframe 外链播放器无法稳定提供音乐播放器需要的播放、暂停、继续、进度条和连播能力。

新路线改为：

```text
目录链接
-> 用户主动保存为本地音频
-> 原生音频播放器播放
-> 组成本地听单
```

这条路线的产品边界更清楚：广场和歌单首先是目录，真正播放发生在用户主动保存到本地之后。

## 4. 核心用户

### 目标用户

1. 在 B 站高频听 AI 歌曲、AI 翻唱、AI 风格改编的用户
2. 喜欢把内容整理成播放单、连续听的人
3. 需要把视频内容转成“可听内容”的用户
4. 可能把内容作为学习播放单使用的人

### 核心需求

- 统一整理分散链接
- 建立自己的本地听单
- 能真正播放、暂停、继续、拖动进度条
- 能连续播放多首
- 能删除不相关内容
- 能清理本地缓存
- 能看到标题、封面、来源、作者、时长和轻量 AI 信息

## 5. MVP 目标

MVP 只验证一个核心问题：

**用户是否愿意把 B 站 AI 音乐目录保存成本地音频，并把它作为听歌入口持续使用。**

### P0 验证链路

```text
B 站单条链接
-> 解析标题/封面/作者/时长
-> 用户点击保存为本地音频
-> 加入实验听单
-> 原生播放器播放
-> 支持暂停、继续、拖动进度、连续播放
```

### P1 验证链路

```text
B 站收藏夹/播放列表链接
-> 解析为候选目录
-> 用户删除非歌曲/非目标内容
-> 批量保存为本地音频
-> 组成听单连续播放
```

## 6. 内容源策略

### P0：B 站

- 大陆版默认公开来源
- 支持单条链接
- 支持收藏夹/播放列表解析为候选目录
- 支持用户主动保存为本地音频
- 支持本地听单播放

### P1：YouTube

- 技术层预留
- 国际版/海外版本可公开
- 大陆版默认不露出
- 可通过实验功能或灰度配置开启
- 不作为大陆 MVP 成败关键

### P2：TikTok

- 可做实验来源
- 只验证单条链接
- 不承诺稳定性

### P3：抖音

- 高风险实验来源
- 不作为 MVP P0/P1
- 不承诺收藏夹/批量导入

### 暂不做

- 小红书
- 快手
- 微信视频号
- Instagram/Facebook/X 等强风控来源

## 7. 核心功能

### 7.1 单条导入

- 用户粘贴 B 站链接
- 系统解析 BVID、标题、封面、UP 主、时长
- 用户确认保存为本地音频
- 保存完成后加入当前听单
- 可立即播放

### 7.2 收藏夹/播放列表导入

- 用户粘贴 B 站收藏夹或播放列表链接
- 系统解析为候选列表
- 用户可以删除非歌曲、非学习内容或不想缓存的条目
- 用户确认后逐首保存为本地音频
- 保存完成后生成一组听单

支持的链接形态包括：

- `favlist?fid=...`
- `media_id=...`
- `mlid=...`
- `/lists/...`
- `/medialist/detail/ml...`
- `/list/ml...`

### 7.3 本地听单

- 多首本地音频组成播放单
- 当前曲目结束后自动播放下一首
- 支持手动切换曲目
- 支持从听单移除条目
- 支持清空本地缓存

### 7.4 播放器

- 使用原生音频能力
- 支持播放
- 支持暂停
- 支持继续播放
- 支持拖动进度条
- 支持读取真实时长
- 支持本地封面展示

### 7.5 目录广场

MVP 阶段广场只做目录，不直接播放。

广场内容可以包括：

- 编辑精选目录
- 最新收录目录
- 热门收藏目录
- AI 音乐专题目录
- 学习播放单目录

用户如果想使用目录内内容，需要先保存成本地音频。

### 7.6 轻量 AI 信息

首期只保留结构化字段：

- 风格标签
- 内容类型
- 短备注

后置字段：

- 情绪标签
- 模型字段
- Prompt 信息
- 创作者补充说明

## 8. 本地缓存策略

### 原则

- 用户主动触发
- 本地个人使用
- 不公开分发
- 不上传到平台云端供他人播放
- 可删除

### 用户可见规则

- 未缓存内容显示为“目录项”
- 缓存中内容显示进度状态
- 已缓存内容可播放
- 缓存失败要明确显示原因
- 用户可以删除单首缓存
- 用户可以清空整个听单缓存

### 后续增强

- 后台缓存队列
- 缓存失败重试
- 网络恢复后继续
- 批量缓存进度
- 存储空间管理

## 9. 发现页与广场边界

广场不做“公开播放站”，只做目录发现。

首页可以展示：

- 编辑精选目录
- AI 翻唱热门目录
- 最新收录
- 用户公开目录
- 学习播放单目录

但点击内容后，不直接播放，而是进入详情或目录页，引导用户：

```text
保存为本地音频后收听
```

学习播放单可以成立，但不作为首页主心智。它可以作为用户自定义使用方式存在。

## 10. 账号与创作者策略

### 验证目标

验证的是来源账号控制权，不是法律意义上的版权归属。

### 首期策略

- 账号绑定后，可以把该来源账号下已收录作品关联到创作者页
- 创作者可补充 AI 信息和短备注
- 创作者可申请专题目录

### 后置策略

- 创作者上传授权音频
- 创作者官方听单
- 创作者页增强

## 11. 审核、合规与风险边界

### 核心风险

新路线从“来源播放”变成“本地音频缓存”，产品体验更成立，但风险更清晰也更高。

主要风险：

- 平台条款风险
- 版权投诉风险
- App Store 审核风险
- 用户误解为公开下载器
- 商业化后风险提升

### 风险控制

- 不使用“下载器”“转 MP3”等主文案
- 使用“保存为本地收听”“本地听单”“个人离线收听缓存”
- 不提供音频文件外发功能
- 不生成公开音频播放链接
- 不在广场直接播放缓存音频
- 默认不自动批量缓存
- 批量缓存必须由用户确认
- 支持清理缓存

## 12. 商业化方向

MVP 不做商业化。

后续可能方向：

- 高级听单管理
- 更强播放器能力
- 更大的本地缓存管理
- 创作者/策展人增强页
- 专题目录合作

不建议早期对“保存本地音频”直接收费，否则风险显著升高。

## 13. 成功指标

### 核心指标

- 单条解析成功率
- 本地音频保存成功率
- 首次播放成功率
- 暂停/继续使用率
- 进度拖动使用率
- 人均听单曲目数
- 单次会话连续播放曲目数
- 本地缓存保留率
- 7 日回访率

### 关键判断

- 用户是否愿意保存成本地音频
- 用户是否会把它当成听歌入口
- 用户是否会整理多个目录为听单
- 用户是否会从 B 站收藏夹筛选出真正想听的内容

## 15. 结论

产品路线应从：

```text
可播放目录型 AI 音乐工具
```

调整为：

```text
AI 音乐目录 + 用户主动本地音频缓存 + 本地听单播放器
```

如果本地缓存播放体验稳定，产品成立；如果只能依赖外链 iframe 播放，产品不成立。
