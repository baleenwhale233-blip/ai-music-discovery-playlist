# Product

## 产品定位

这是一个面向大陆用户的 Mobile Web First 视频转音频目录与本地听单工具。

用户从 B 站等来源添加视频或列表链接，系统解析标题、封面、作者、时长等目录信息。用户可以把目录条目整理成听单，并在主动确认后缓存为个人本地音频资产。播放发生在缓存完成之后，基于全局播放器和真实 media element 支持播放、暂停、继续、拖动进度、连续播放和播放模式切换。

## 目标用户

- 高频在 B 站收听歌曲、翻唱、风格改编、学习视频和长内容的用户。
- 希望把分散视频链接整理成可连续收听目录的人。
- 需要把视频内容转成个人收听资产的用户。
- 会把非歌曲内容整理成学习播放单的用户。

## Alpha 目标

Alpha 阶段验证一个核心问题：用户是否愿意把来源目录整理成个人听单，并主动缓存为可连续播放的本地音频资产。

验收重点：

- B 站链接可以解析为稳定目录信息。
- 用户可以从候选条目中筛选内容。
- 用户可以创建、编辑和发布个人听单目录。
- 用户可以收藏公开听单目录，并区分自己创建和收藏的听单。
- 用户可以主动缓存条目。
- 已缓存条目可以通过全局 mini player 和完整播放器连续收听。

## 核心链路

```text
添加链接
-> 解析目录
-> 整理听单
-> 用户主动缓存
-> 本地音频资产
-> 真播放器播放
```

P0 链路聚焦 B 站单条链接和 B 站收藏夹/播放列表候选解析。P1 链路扩展批量筛选、批量缓存、缓存失败重试和听单管理。

## 当前范围

- Mobile Web Alpha 产品入口。
- Alpha 登录。
- B 站单条链接解析。
- B 站收藏夹、播放列表、合集类链接解析为候选目录。
- 创建听单草稿。
- 从链接添加候选条目。
- 发布本地听单目录。
- 管理自己创建的听单目录。
- 收藏和取消收藏公开听单。
- 用户主动缓存条目为个人本地音频资产。
- 全局 mini player 和完整播放器播放已缓存音频。
- 本地缓存清理。
- 内部后台和调试入口。

## 当前不包含

- 公开音频托管。
- 向其他用户公开分享个人缓存文件。
- 用户上传型音乐平台。
- 重社区、评论、关注流。
- 版权归属认证平台。
- 把来源播放器 iframe/WebView 当主播放方式。
- 完整原生 App 分发。

## 内容源策略

B 站是 Alpha 的 P0 来源：

- 支持单条视频链接。
- 支持收藏夹、播放列表、合集类链接。
- 解析标题、封面、作者、时长和来源 ID。
- 将来源内容统一归档为目录候选。

YouTube、TikTok、抖音等来源保留在技术和实验层，不作为大陆 Alpha 的默认公开来源。

## 本地缓存与播放策略

本地缓存遵守这些产品规则：

- 用户主动触发缓存。
- 缓存资产属于当前用户和当前后端环境。
- 缓存文件可删除。
- 缓存完成后才进入播放器。
- 播放器使用真实 media element。
- 音频接口支持 Range 响应，便于拖动进度和继续播放。
- mini player 固定在底部导航栏上方；完整播放器负责队列、上一首、下一首、列表循环、单曲循环和随机播放。

平台不把个人缓存文件作为公开媒体资源分发。

## 风险边界

- 来源平台条款和可用性风险。
- 转换节点稳定性。
- 转换成本和并发控制。
- 本地缓存空间管理。
- 封面和音频代理的访问稳定性。
- Alpha 登录和个人资产隔离。

产品策略通过用户主动缓存、个人资产边界、目录优先展示和缓存清理能力降低风险。

## 成功指标

- 新用户可以在移动端完成登录、添加链接、解析、筛选、发布和缓存。
- 单条链接解析成功率稳定。
- 收藏夹/播放列表候选解析可以在可接受时间内返回。
- 已缓存音频的播放、暂停、继续、拖动进度和连续播放稳定。
- 用户愿意持续维护个人听单，而不是只做一次性转换。
